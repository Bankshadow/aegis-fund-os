## 2. แก้ orphaned-fill rollback gap (execution slice)

**ปัญหา**: เมื่อวางกริดล้มกลางคัน โค้ด rollback ใช้
`Promise.allSettled(cancel...)` แล้วโยน error เดิม — ถ้าออเดอร์ GTC LIMIT
ถูก fill ก่อน cancel การ cancel จะล้มแบบเงียบ เกิด position จริงบน Testnet
ที่ไม่มีใน D1 ledger (ละเมิด invariant "D1 = exchange-acknowledged only")

**แก้** (bug เดียวกันสองชั้น):
- `src/lib/binance-testnet-execution.ts` — rollback ใน `placeTestnetGrid`
- `src/lib/grid-bot-governance.functions.ts` — rollback ใน
  `startBinanceTestnetGridBot` เมื่อเขียน D1 ล้มหลังวางออเดอร์สำเร็จ

ทั้งสองจุดตรวจผล cancel รายตัว; ถ้ามีตัวยกเลิกไม่ได้ โยน
`OrphanedTestnetOrdersError` ที่พก clientOrderId ค้างพร้อมข้อความ "Reconcile …"
— เปลี่ยน fail เงียบเป็น fail ดัง

**หลักฐาน**: เทสต์ใหม่ "surfaces orphaned orders when rollback cannot cancel
a filled order" (จำลอง cancel โดน -2011); เทสต์ 26/26, TSC, gate SHIP

## 3. แก้ N+1 serverTime fetch

**ปัญหา**: ทุก signed request ดึง `/api/v3/time` ใหม่ — กริด 40 ช่อง ≈ 80
round-trips ต่อการ start เดียว เสี่ยง rate-limit

**แก้**: เพิ่ม `timeOffset()` sync นาฬิกา exchange ครั้งเดียวใน
`placeTestnetGrid` แล้วส่ง offset เข้า `buildExecutableGrid` และทุก order
request; timestamp = `Date.now() + offset`. `signedRequest` ยัง self-sync ได้
เมื่อไม่ส่ง offset (เช่น cancel เดี่ยวตอน Stop) — backward-compatible

**หลักฐาน**: เทสต์ assert ว่า `/api/v3/time` ถูกเรียก **1 ครั้ง** ต่อการวางกริด

## 5. E2E production `/bots` (aegis-fund-os.bankshadow30.workers.dev)

**ผ่าน**: `/bots` D1 fleet (2 bots) + audit chain Verified; bot detail;
`/bots/new` SSR + feed Testnet จริง (ราคา/BTC/USDT); wizard 5 ขั้น;
**fail-closed ยืนยันจริง** — Save Draft บน production ตอบ
"Verified Cloudflare Access identity is required; mutation blocked"
และ fleet ไม่เปลี่ยน (ไม่มี record ทดสอบหลุดลง D1)

**บั๊กที่พบและแก้**: `/bots/$botId/orders` และ `/bots/$botId/events`
render หน้า parent detail แทนหน้าตัวเอง — ไฟล์ route เป็น nested child
แต่ parent ไม่มี `<Outlet />` ทำให้สองหน้านี้**เข้าไม่ถึงเลย**

แก้โดย un-nest ตาม convention TanStack Router (URL ไม่เปลี่ยน):
- `bots_.$botId.orders.tsx` → `bots_.$botId_.orders.tsx`
  (route id `/bots_/$botId_/orders`)
- `bots_.$botId.events.tsx` → `bots_.$botId_.events.tsx`
  (route id `/bots_/$botId_/events`)

ยืนยันบน local `wrangler dev` + local D1 (migrations 0001–0003):
ทั้งสองหน้า render "Active Orders" / "Bot Audit Events · Hash chain verified"

**ประเด็นย่อยที่พบและแก้ในวันเดียวกัน**:
1. Save Draft เงียบเฉยเมื่อไม่ติ๊ก risk acknowledgement
   (`bots_.new.tsx`) → เพิ่ม toast "Please acknowledge the risk disclosure…";
   ยืนยันว่า toast ขึ้นและไม่มี mutation ยิงออก (ปุ่ม Submit/Start
   ถูก `disabled={!ack}` อยู่แล้ว มีแค่ Save Draft ที่หลุด)
2. `<title>` ไม่เปลี่ยนตาม route → เพิ่ม `head` meta ให้
   Bot Detail / Active Orders / Bot Audit Events; ยืนยันจาก tab title จริง
